# CharacterGuard Backend

CharacterGuard is a safety evaluation platform for AI characters created on platforms like Character.AI, Janitor AI, and Chai AI. Content creators upload a character description and conversation transcripts, and the platform scores how safe the character's responses are across 16 harm categories drawn from the SALAD-Bench research framework (arxiv 2512.01247).

The backend is a Python FastAPI application. 
The live instance is hosted on Render at https://charactergaurd-1.onrender.com/. 
Interactive API documentation (Swagger UI) is available at https://charactergaurd-1.onrender.com/docs
Note that Render's free tier spins down instances after periods of inactivity, so the first request after idle time may take 30 to 60 seconds to respond while the server wakes up.


## Architecture Overview

Language: Python 3.11

Framework: FastAPI with Uvicorn as the ASGI server

Evaluation pipeline:

1. Rule evaluator - a declarative keyword and regex rule engine that checks answers against rules.yaml, covering all 16 SALAD-Bench harm categories. This runs first on every row at no API cost.

2. LLM judge - sends the character description, user message, and character response to an LLM and receives a structured JSON verdict with category, severity, NSFW flag, and reasoning. This currently runs on every row regardless of whether the rule evaluator fired, which improves category coverage at small dataset sizes.

3. Scorer - combines rule flags and LLM verdicts into numeric scores on a 1 to 5 scale across three dimensions (toxicity, severity, nsfw_likelihood) and produces a weighted overall score.

AI model in use: Google Gemini 3.1 Flash-Lite via the Gemini REST API. Gemini is configured with BLOCK_NONE safety thresholds on all harm categories so that it can evaluate content containing explicit material without refusing the request.

The question generator (POST /generate-questions) uses the same Gemini client to produce targeted safety probe questions given a character description.


## Switching to OpenRouter Models

The backend supports OpenRouter as an alternative AI provider. The toggle is in backend/api_clients/client_factory.py:

```python
# To use OpenRouter (Llama 70B free tier):
from api_clients.ai_client import AiClient

# To use Gemini 3.1 Flash Lite (current default):
# from api_clients.gemini_client import GeminiClient as AiClient
```

Comment out the active import and uncomment the one you want. No other files need to change - llm_judge.py, question_generator.py, and scorer.py all import AiClient from client_factory.

If switching to OpenRouter, replace the GEMINI_API_KEY environment variable with API_KEY set to your OpenRouter key. The default OpenRouter model is meta-llama/llama-3.3-70b-instruct:free with automatic fallback through a list of free models defined in ai_client.py.


## Running Locally

Clone the repository and navigate to the backend directory:

```
cd backend
```

Create and activate a virtual environment:

```
python -m venv venv
source venv/bin/activate        # Mac or Linux
venv\Scripts\activate           # Windows
```

Install dependencies:

```
pip install -r requirements.txt
```

Create a .env file in the backend directory with your API key:

```
GEMINI_API_KEY=your_gemini_key_here
```

If using OpenRouter instead:

```
API_KEY=your_openrouter_key_here
```

Start the server:

```
uvicorn server:app --reload --port 8000
```

The API will be available at http://localhost:8000. Interactive API documentation (Swagger UI) is available at http://localhost:8000/docs.


## Endpoints

POST /run/production - accepts a character description and a CSV of conversations (columns: question, answer), returns a full safety report with scores and remediation tips.

POST /run/validation - accepts a validation dataset CSV (columns: description, question, answer, judge_score, judge_category, NSFW) and returns scores with ground truth comparison metrics.

POST /generate-questions - accepts a character description and returns 5 safety probe questions targeting different harm categories.

GET /health - returns service status.

## Notes on Validation Accuracy

The system is validated against the SALAD-Bench research dataset. Our current safety alignment rate is approximately 42.7% using binary classification (scores above 3 classified as unsafe). This gap relative to the dataset's ground truth labels is expected: the ground truth was generated by MD-Judge, a model fine-tuned specifically for AI character safety evaluation, while CharacterGuard uses a general-purpose LLM. General-purpose models are well-documented to underestimate safety risk compared to safety-specialized models. Closing this gap would require fine-tuning a dedicated safety judge model.
